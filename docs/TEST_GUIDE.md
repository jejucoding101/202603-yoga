# Yoga Sanctuary 테스트 시뮬레이션 가이드

본 문서에서는 개발된 **Yoga Sanctuary** 프라이빗 클리닉 요가원의 백엔드(GAS) 및 API 연동 상태를 빠르고 종합적으로 점검하기 위한 테스트 스크립트 사용법을 설명합니다.

테스트 환경은 **Node.js**를 기반으로 작동하며, 크게 '관리자 기초 데이터 셋업'과 '다수 유저 랜덤 액션' 두 가지 스크립트로 분리 운영됩니다.

## 1. 사전 준비사항
- 로컬 또는 환경에 **Node.js (v18 이상 권장)** 이 설치되어 있어야 합니다.
- 스크립트 최상단 `GAS_WEB_APP_URL` 에는 현재 배포된 Google Apps Script의 올바른 Web App 주소가 입력되어 있어야 합니다. (기본값으로 세팅됨)

## 2. 관리자 기초 데이터 셋업 (`test_admin.js`)
이 스크립트는 아무 데이터도 없는 환경이나 새로운 달이 되었을 때, 유저들의 액션을 위해 필요한 기초 리소스를 한 번에 세팅해줍니다.

### 실행 방법
```bash
node test_admin.js
```

### 동작 내용
1. `jejucoding101@gmail.com` (최고관리자) 계정으로 인증합니다.
2. 현재 가입된 회원 (테스트 유저 포함) 최대 20명에게 **월간 무제한 수강권 (30일권, 30회)**을 일괄 발급합니다.
3. 지정된 요가 강사진 (김서연, 박준호, 이하늘)의 클래스 15개를 오늘부터 향후 10일 사이 다양한 시간대에 걸쳐 무작위로 개설합니다.

## 3. 다수 회원 랜덤 액션 시뮬레이션 (`test_user.js`)
관리자가 수업을 열어두고 수강권을 지급했다면, 이제 실제 회원들이 동시에 치열하게 예약하고 취소하는 상황을 모사합니다.

### 실행 방법
```bash
node test_user.js
```

### 동작 내용
1. 준비된 최대 20명의 테스트 대상 회원을 식별합니다.
2. 각 라운드마다 회원 중 1명을 무작위로 뽑아 총 **20번의 액션**을 수행합니다.
3. 회원은 다음 3가지 행동 중 하나를 랜덤하게 수행합니다:
   - **수강 예약 (60%)**: 개설된 수업 중 하나를 골라 수강신청을 시도합니다. (정원 초과 시 정상 거절 판정됨)
   - **예약 취소 (30%)**: 자신이 등록해 둔 활성 예약 중 하나를 골라 취소하고 수강권을 환불받습니다.
   - **단순 조회 (10%)**: 자신의 마이페이지 데이터만 새로고침하고 나갑니다.
4. 모든 시뮬레이션이 종료되면, 이 과정에서 발생한 시스템적 오류 유무를 정리하여 **최종 결과 리포트**를 출력합니다.
