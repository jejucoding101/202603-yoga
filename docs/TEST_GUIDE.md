# Yoga Sanctuary 테스트 시뮬레이션 가이드

본 문서에서는 개발된 **Yoga Sanctuary** 프라이빗 클리닉 요가원의 백엔드(GAS) 및 API 연동 상태를 빠르고 종합적으로 점검하기 위한 테스트 스크립트 사용법을 설명합니다.

테스트 환경은 **Node.js**를 기반으로 작동하며, 크게 '관리자 기초 데이터 셋업'과 '다수 유저 랜덤 액션' 두 가지 스크립트로 분리 운영됩니다.

## 1. 사전 준비사항
- 로컬 또는 환경에 **Node.js (v18 이상 권장)** 이 설치되어 있어야 합니다.
- 스크립트 최상단 `GAS_WEB_APP_URL` 에는 현재 배포된 Google Apps Script의 올바른 Web App 주소가 입력되어 있어야 합니다. (기본값으로 세팅됨)

## 2. 🌟 권장 테스트 시나리오 플로우
Yoga Sanctuary 웹사이트 오픈 전, 대규모 트래픽과 예약 비즈니스 로직을 완벽하게 검증하기 위해 다음 순서로 스크립트를 구동하는 것을 강력히 권장합니다.

1. **트래픽/회원 생성 (`node test_traffic.js`)**: 터미널 1에서 실행하여 20명의 테스트 회원을 가입시키고 접속 기록(로그인 카운트)을 백그라운드에서 무한히 올립니다. (테스트 끝날 때까지 켜둠)
2. **어드민 환경 셋업 (`node test_admin.js`)**: 터미널 2에서 1회 실행하여 개설된 20명의 회원에게 수강권을 지급하고, 15개의 강의를 캘린더에 무작위 생성합니다.
3. **유저 혼돈 모의 (`node test_user.js`)**: 같은 터미널 2에서 실행하여, 방금 가입된 20명의 회원이 생성된 15개 강의를 두고 치열하게 예약 및 취소 버튼을 누르는 경쟁 상황(총 20라운드)을 시뮬레이션 합니다.

## 3. 테스트 회원 대량 가입 (`test_traffic.js`)
본격적인 테스트 전, 테스트용 가상 계정 20명을 요가원 DB(Google Sheets)에 자동으로 가입시키는 스크립트입니다. 이미 가입된 회원이더라도 중복 처리를 우회합니다.

### 실행 방법
```bash
node test_traffic.js
```

### 동작 내용
1. `testuser1@yoga.com` 부터 `testuser20@yoga.com` 까지, 총 20명의 테스트 회원을 Google Sheets `Users` 시트에 순차적으로 추가합니다.
2. 가입 완료 후, 해당 스크립트는 종료하지 않고 무한 루프를 돌며 무작위로 계속 로그인을 수행하여 **앱 로그인 카운트 및 최근 접속일** 트래픽을 올려주는 현황을 생성합니다.

## 4. 관리자 기초 데이터 셋업 (`test_admin.js`)
이 스크립트는 아무 데이터도 없는 환경이나 새로운 달이 되었을 때, 유저들의 액션을 위해 필요한 기초 리소스를 한 번에 세팅해줍니다.

### 실행 방법
```bash
node test_admin.js
```

### 동작 내용
1. `jejucoding101@gmail.com` (최고관리자) 계정으로 인증합니다.
2. 현재 가입된 회원 (테스트 유저 포함) 최대 20명에게 **월간 무제한 수강권 (30일권, 30회)**을 일괄 발급합니다.
3. 지정된 요가 강사진 (김서연, 박준호, 이하늘)의 클래스 15개를 오늘부터 향후 10일 사이 다양한 시간대에 걸쳐 무작위로 개설합니다.

## 5. 다수 회원 랜덤 액션 시뮬레이션 (`test_user.js`)
관리자가 수업을 열어두고 수강권을 지급했다면, 이제 실제 회원들이 동시에 치열하게 예약하고 취소하는 상황을 모사합니다.

### 실행 방법
```bash
node test_user.js
```

### 동작 내용
1. 준비된 최대 20명의 테스트 대상 회원을 식별합니다.
2. 각 라운드마다 회원 중 1명을 무작위로 뽑아 총 **20번의 액션**을 수행합니다.
3. 회원은 다음 3가지 행동 중 하나를 랜덤하게 수행합니다:
   - **수강 예약 (60%)**: 개설된 수업 중 하나를 골라 수강신청을 시도합니다. (정원 초과 시 정상 거절 판정됨)
   - **예약 취소 (30%)**: 자신이 등록해 둔 활성 예약 중 하나를 골라 취소하고 수강권을 환불받습니다.
   - **단순 조회 (10%)**: 자신의 마이페이지 데이터만 새로고침하고 나갑니다.
4. 모든 시뮬레이션이 종료되면, 이 과정에서 발생한 시스템적 오류 유무를 정리하여 **최종 결과 리포트**를 출력합니다.

## 6. 테스트 종료 후 데이터 통째로 초기화하는 방법 (중요)
스크립트를 구동하고 나면 수많은 예약 기록과 회원, 클래스가 Google Sheets에 채워집니다. **실제 홈페이지 운영을 시작하기 전 이 데이터들을 모두 지우고 초기화하려면 다음 규칙을 따라야 합니다.**

1. Google Sheets (`Yoga_Sanctuary_DB`) 접속
2. `Users`, `Passes`, `Classes`, `Bookings` 4개의 시트를 순회합니다.
3. **각 시트의 1행(A1, B1 등 컬럼 제목(헤더)이 있는 줄)은 절대 지우면 안 됩니다!**
   - 백엔드 스크립트가 1행의 제목을 기준으로 데이터를 파싱하므로, 1행이 지워지면 API가 고장납니다.
4. **2행부터 그 아래 데이터가 작성된 모든 행을 드래그하여 삭제** (단순 Delete키(내용 지우기) 혹은 우클릭 - 행 삭제) 하시면 됩니다.
5. 위 방법으로 4개의 시트 데이터를 모두 비우면 웹사이트는 완벽한 **초기 상태(0명, 0개 수업)**로 돌아갑니다.
